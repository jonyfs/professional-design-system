## Summary

Ship three production-grade, accessible HTML + Tailwind CSS primitives
(Radio, Select, Toggle/Switch), completing the Forms, Validation & Inputs
section of the Component Catalog alongside feature 001's Text Input and
Checkbox. Radio and Select reuse feature 001's exact token vocabulary
(verified below, not assumed). Toggle reuses the ratified track/dot pattern
but adds a boundary ring ‚Äî a real perceivability gap in the ratified
pattern, found during this plan's research and fixed with an already-
ratified token, no amendment required. No new project tooling: this reuses
001's Vite/Tailwind/Playwright/audit-script scaffold entirely.

## Technical Context

**Language/Version**: HTML5, Tailwind CSS 3.4.x (existing scaffold), vanilla
JS not required ‚Äî Radio/Select/Toggle all use native pseudo-classes and
native form-control behavior (`:checked`, grouped `name` radio semantics,
native `<select>` option list).

**Primary Dependencies**: None new ‚Äî reuses `tailwindcss@^3.4`, `vite@^8`,
`@playwright/test`, `@axe-core/playwright`, `wcag-contrast` already in
`package.json` from feature 001.

**Storage**: N/A

**Testing**: Same Playwright visual regression (320/768/1024/1440,
Chrome/Firefox/Safari) + `@axe-core/playwright` pattern as feature 001,
extending the existing `tests/e2e/a11y-helper.ts`. **Critical process
correction from feature 001's CI incident**: new visual regression
baselines MUST be generated by triggering `update-snapshots.yml`
(`workflow_dispatch`) and downloading its artifact ‚Äî NOT locally, and NOT
via a local Docker Playwright container. Feature 001 discovered that even
`mcr.microsoft.com/playwright` run locally on a Mac's Docker Desktop VM is
not bit-identical to the real `ubuntu-latest` GitHub-hosted runner (subtle
font/antialiasing diffs caused 84 false failures). Only the actual CI
runner is guaranteed to produce a matching baseline.

**Target Platform**: Web ‚Äî current stable evergreen browsers, responsive
320px‚Äì1920px. Same as feature 001.

**Project Type**: Single-project static frontend component library
(unchanged from feature 001 ‚Äî no new structure).

**Performance Goals**: Same CSS bundle budget as feature 001 (<15kb
gzipped, microsite budget) ‚Äî three more components must not meaningfully
move this; verified in the Constitution Check re-evaluation post-design
and again at build time.

**Constraints**: Same as feature 001 (Tailwind-only architecture, zero raw
palette classes, full interactive state coverage, English-only artifacts).

**Scale/Scope**: 3 primitives (Radio, Select, Toggle), single feature
slice, reuses all existing tooling.
